# Security Policy

## Supported versions

The latest `main` branch is supported.

## Reporting a vulnerability

Please report vulnerabilities privately to maintainers before public disclosure.

Include:

- Affected component
- Reproduction steps
- Potential impact
- Suggested mitigation (if available)

## Operational hardening

- Use dedicated RPC credentials and IP allowlists.
- Restrict DB network access and rotate credentials.
- Keep confirmations/reorg buffer tuned for your target chain.
