# Contributing

## Development workflow

1. Fork and create a feature branch.
2. Add or update tests for all behavior changes.
3. Run local checks: `mvn clean test`.
4. Open a PR with clear summary and risk notes.

## Commit style

- Keep commits focused and atomic.
- Use imperative tense in commit messages.

## Code standards

- Java 17, Spring Boot 3 conventions.
- Keep services single-responsibility.
- Add structured logs for chain-facing operations.
